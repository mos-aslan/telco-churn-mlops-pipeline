# چارچوب گزارش نهایی Word

این فایل فقط چارچوب گزارش نهایی است. گزارش Word نهایی در مرحله آخر نوشته می‌شود.

## ساختار پیشنهادی گزارش

1. عنوان پروژه
2. معرفی پروژه و هدف
3. معرفی دیتاست Telco Customer Churn
4. نسخه‌بندی داده‌ها: v1, v2, v3
5. پیش‌پردازش داده‌ها
6. Feature Engineering
7. مدل‌های استفاده‌شده
8. روش ارزیابی: Train/Validation/Test و Cross Validation
9. ثبت آزمایش‌ها در MLflow
10. مقایسه مدل‌ها و نتایج
11. انتخاب مدل نهایی
12. استقرار مدل با Docker
13. اجرای Pipeline
14. GitHub Repository و Commitها
15. تصاویر و شواهد
16. نتیجه‌گیری

## تصاویر مورد نیاز در گزارش

- تصاویر MLflow
- تصاویر Docker
- تصاویر GitHub و Commit History
- تصاویر اجرای Pipeline
- نمودارهای مهم EDA و مقایسه مدل‌ها
